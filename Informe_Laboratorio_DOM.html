Resumen

La práctica de laboratorio abordó el desarrollo de una aplicación web para el registro dinámico de estudiantes, empleando manipulación del Document Object Model (DOM) mediante JavaScript vanilla. El sistema implementó un formulario de entrada, una tabla dinámica con acciones por fila, un modal Bootstrap para la visualización de datos en formato JSON, funciones de edición y eliminación de registros, filtros inline de estilo Excel por columna, validación de entrada alfanumérica y la integración de las librerías SweetAlert2 y Toastify para mejorar la experiencia visual del usuario. El trabajo permitió consolidar conocimientos sobre acceso a elementos HTML, creación y modificación de nodos, manejo de eventos, uso de arreglos de objetos y presentación de datos serializados.

Objetivos

Objetivo General

Analizar las capacidades de manipulación del DOM que ofrece JavaScript para el desarrollo de una aplicación web interactiva de registro de estudiantes, comprendiendo cómo las modificaciones dinámicas del documento permiten construir interfaces funcionales sin necesidad de recargar la página.

Objetivos Específicos

Evaluar los mecanismos de acceso y modificación de elementos HTML mediante métodos como getElementById, createElement, appendChild e innerHTML, aplicándolos al construcción de formularios interactivos y tablas que se actualizan en tiempo real a partir de la entrada del usuario.

Determinar la utilidad de los arreglos de objetos como fuente estructurada de datos en JavaScript, implementando operaciones de CRUD (crear, leer, actualizar, eliminar) sobre un conjunto de registros de estudiantes y verificando la integridad de la información almacenada.

Integrar librerías externas como SweetAlert2 y Toastify en un entorno web, evaluando cómo estas herramientas mejoran la experiencia visual del usuario respecto a las alertas nativas del navegador y las funciones predeterminadas de Bootstrap.

Introducción

El Modelo de Objetos del Documento, conocido como DOM por sus siglas en inglés (Document Object Model), constituye la representación estructurada que el navegador construye a partir del código HTML de una página web. Flanagan (2011) describe el DOM como una interfaz de programación que permite a los lenguajes de scripting como JavaScript acceder y modificar el contenido, la estructura y el estilo de los documentos web de manera dinámica. Cuando el navegador carga un archivo HTML, genera un árbol de objetos donde cada etiqueta se convierte en un nodo que puede ser leído, alterado, creado o eliminado por el programador. Mozilla Developer Network (2024) señala que el DOM no es parte del lenguaje JavaScript en sí, sino una API independiente que el lenguaje utiliza para interactuar con el contenido de la página. Crockford (2008) aporta que el manejo del DOM es una de las habilidades fundamentales para el desarrollo web moderno, ya que permite construir interfaces reactivas sin depender de recargas completas del servidor. W3C (2021) define el estándar del DOM como un modelo jerárquico donde cada nodo del árbol posee relaciones de padre, hijo y hermano, lo cual facilita la navegación programática del documento. Esta práctica se centra en la aplicación directa de estos conceptos para construir un sistema funcional de registro de estudiantes.

Marco Teórico

El Modelo de Objetos del Documento (DOM)

El DOM se define como una representación en forma de árbol del documento HTML donde cada elemento, atributo y texto se convierte en un nodo accesible por programación. Flanagan (2011) explica que cuando el navegador parsea el HTML, genera una estructura jerárquica donde el nodo raíz es el objeto document, y desde éste se accede a todos los elementos descendientes. Mozilla Developer Network (2024) precisa que el DOM Level 2 y el DOM Level 3 extienden las capacidades del modelo original, permitiendo la manipulación de estilos CSS, la escucha de eventos y la modificación del contenido de manera estándar. Duckett (2011) describe que el DOM permite tres operaciones fundamentales: buscar elementos existentes, crear nuevos elementos y modificar o eliminar elementos ya presentes en el documento. Estas operaciones son la base de toda interacción dinámica en el navegador.

Eventos en JavaScript

Los eventos constituyen señales que el navegador genera cuando ocurre una acción sobre un elemento del DOM, ya sea una interacción del usuario como un clic, un movimiento del cursor o una pulsación de tecla. Flanagan (2011) define un evento como un objeto que representa una ocurrencia que sucede en el sistema y que el navegador notifica al código JavaScript mediante un modelo de propagación. El evento click es uno de los más utilizados en el desarrollo web, pues permite ejecutar una función cada vez que el usuario presiona un botón o un enlace. Crockford (2008) indica que el método addEventListener es la forma recomendada de asociar funciones a eventos, reemplazando las propiedades en línea que generan código menos mantenible. El modelo de eventos de DOM incluye tres fases: captura, objetivo y propagación, lo que permite un control preciso sobre qué función responde en qué momento.

Métodos de Manipulación del DOM

JavaScript ofrece un conjunto de métodos para interactuar con los nodos del DOM. Los más frecuentes incluyen getElementById, que localiza un elemento por su atributo id; createElement, que genera un nuevo nodo del tipo especificado; appendChild, que inserta un nodo como hijo al final de un elemento padre; y textContent e innerHTML, que permiten leer o escribir el contenido textual o HTML de un elemento. Mozilla Developer Network (2024) describe que textContent opera sobre el contenido de texto plano, mientras que innerHTML interpreta el contenido como markup HTML. Además, las propiedades className y classList permiten modificar las clases CSS aplicadas a un elemento, lo cual resulta esencial para cambiar estilos dinámicamente. La combinación de estos métodos facilita la construcción de interfaces que responden en tiempo real a las acciones del usuario.

Librerías Externas: SweetAlert2 y Toastify

Las librerías externas aportan funcionalidades preconstruidas que complementan las capacidades nativas del navegador. SweetAlert2 es un recurso de código abierto que reemplaza la función alert() nativa por una ventana modal estilizada, permitiendo personalizar títulos, textos, iconos y botones de acción. SweetAlert2 (2024) documenta que su API permite configurar confirmaciones interactivas con animaciones y soporte para promesas. Toastify, por su parte, genera notificaciones flotantes que aparecen temporalmente en una esquina de la pantalla, ideales para informar al usuario sobre la ejecución exitosa o fallida de una operación. La integración de estas herramientas mejora la percepción de calidad de la interfaz y reduce la fricción en la experiencia del usuario.

Paso 2: Arreglo global y lógica de registro

Se declaró un arreglo global estudiantesRegistrados que funciona como la fuente principal de datos del sistema. Cada estudiante se almacena como un objeto con tres propiedades: nombre, carrera y semestre. También se declaró una variable editandoIndex que rastrea si el sistema se encuentra en modo de edición o en modo de inserción.

Figura 2. Arreglo global estudiantesRegistrados y declaración de variables.

const estudiantesRegistrados = [];
let editandoIndex = -1;

const btnAgregar = document.getElementById("btnAgregar");
const btnLimpiar = document.getElementById("btnLimpiar");
const btnVerJSON = document.getElementById("btnVerJSON");

La Figura 2 ilustra la inicialización de la estructura de datos y la captura de referencias a los botones principales del formulario. El valor -1 asignado a editandoIndex indica que no hay ningún registro en modo de edición al iniciar la aplicación.

La función agregarEstudiante es el núcleo del sistema de registro. Esta función obtiene los datos del formulario, los valida, determina si el sistema está en modo de edición o inserción, y actualiza tanto el arreglo como la tabla de manera correspondiente.

function agregarEstudiante() {
  const estudiante = obtenerDatosFormulario();

  if (!validarEstudiante(estudiante)) {
    mostrarMensaje("Debe completar todos los campos.", "warning");
    return;
  }

  if (editandoIndex >= 0) {
    estudiantesRegistrados[editandoIndex] = estudiante;
    editandoIndex = -1;
    btnAgregar.textContent = "Agregar estudiante";
    btnAgregar.classList.remove("btn-warning");
    btnAgregar.classList.add("btn-primary");
  } else {
    estudiantesRegistrados.push(estudiante);
  }

  renderizarTablaActual();
  actualizarTotal();
  limpiarFormulario();
}

La Figura 2 (continuación) presenta la lógica dual de la función. Cuando editandoIndex es mayor o igual a cero, la función reemplaza el objeto en esa posición del arreglo; de lo contrario, agrega un nuevo objeto al final con el método push.

Paso 3: Renderizado dinámico de la tabla

La función renderizarTabla recibe una lista de estudiantes y reconstruye completamente el cuerpo de la tabla. Para cada estudiante, se crean nodos tr y td usando document.createElement, se asigna el contenido con textContent y se insertan en el DOM con appendChild. Además, se generan botones de acción (Ver JSON, Editar, Eliminar) con sus respectivos listeners de eventos.

Figura 3. Función renderizarTabla con creación dinámica de nodos.

function renderizarTabla(lista) {
  const cuerpoTabla = document.getElementById("cuerpoTabla");
  cuerpoTabla.innerHTML = "";

  lista.forEach(function (estudiante, indice) {
    const indiceReal = estudiantesRegistrados.indexOf(estudiante);

    const fila = document.createElement("tr");

    const columnaNumero = document.createElement("td");
    columnaNumero.textContent = indiceReal + 1;

    const columnaNombre = document.createElement("td");
    columnaNombre.textContent = estudiante.nombre;

    const columnaAcciones = document.createElement("td");

    const btnVer = document.createElement("button");
    btnVer.textContent = "Ver JSON";
    btnVer.className = "btn btn-sm btn-dark me-1";
    btnVer.addEventListener("click", function () {
      mostrarModalJSONIndividual(estudiante);
    });

    columnaAcciones.appendChild(btnVer);
    fila.appendChild(columnaNumero);
    cuerpoTabla.appendChild(fila);
  });
}

La Figura 3 evidencia el uso del patrón de recreación completa del DOM. Antes de generar las filas, se asigna innerHTML = "" al cuerpo de la tabla para eliminar cualquier nodo existente. Esta técnica, aunque menos eficiente que la actualización selectiva, garantiza consistencia visual y simplifica la lógica cuando la cantidad de registros es moderada.

Paso 4: Modal Bootstrap para visualización JSON

Se integró un modal de Bootstrap para mostrar los datos de un estudiante o del arreglo completo en formato JSON estructurado. La función JSON.stringify convierte los objetos JavaScript en texto legible con sangría, y el resultado se inserta en un elemento pre con estilos de fondo oscuro y texto blanco para simular un visor de código.

Figura 4. Función para abrir el modal con JSON individual.

function mostrarModalJSONIndividual(estudiante) {
  const contenidoJSON = document.getElementById("contenidoJSON");
  contenidoJSON.textContent = JSON.stringify(estudiante, null, 2);

  document.getElementById("modalJSONLabel")
    .textContent = "JSON del estudiante";

  const modal = new bootstrap.Modal(
    document.getElementById("modalJSON")
  );
  modal.show();
}

La Figura 4 muestra cómo se instancia la clase bootstrap.Modal con el identificador del elemento div que contiene el modal. El método show() abre la ventana superpuesta y muestra el contenido JSON formateado.

Paso 5: Edición de registros

Al presionar el botón Editar, los datos del estudiante seleccionado se cargan en los campos del formulario y se activa el modo de edición. El botón principal cambia su texto a “Guardar cambios” y su clase CSS de btn-primary a btn-warning, proporcionando una retroalimentación visual clara sobre el estado del sistema.

Figura 5. Función editarEstudiante con modo de edición.

function editarEstudiante(index) {
  const estudiante = estudiantesRegistrados[index];

  document.getElementById("nombre").value = estudiante.nombre;
  document.getElementById("carrera").value = estudiante.carrera;
  document.getElementById("semestre").value = estudiante.semestre;

  editandoIndex = index;
  btnAgregar.textContent = "Guardar cambios";
  btnAgregar.classList.remove("btn-primary");
  btnAgregar.classList.add("btn-warning");
}

La Figura 5 ilustra la mecánica de edición. La variable editandoIndex almacena la posición del registro dentro del arreglo global. Cuando el usuario presiona “Guardar cambios”, la función agregarEstudiante detecta que editandoIndex es mayor o igual a cero y reemplaza el objeto en esa posición en lugar de agregar uno nuevo.

Paso 6: Eliminación con confirmación

La función eliminarEstudiante utiliza la librería SweetAlert2 para presentar una ventana de confirmación antes de proceder con la eliminación del registro. Esto previene eliminaciones accidentales y proporciona una experiencia visual más pulida que la función confirm() nativa del navegador.

Figura 6. Función eliminarEstudiante con SweetAlert2.

function eliminarEstudiante(index) {
  const estudiante = estudiantesRegistrados[index];

  Swal.fire({
    title: "¿Eliminar estudiante?",
    text: "Se eliminará a " + estudiante.nombre,
    icon: "warning",
    showCancelButton: true,
    confirmButtonColor: "#d33",
    confirmButtonText: "Sí, eliminar",
    cancelButtonText: "Cancelar"
  }).then(function (result) {
    if (result.isConfirmed) {
      estudiantesRegistrados.splice(index, 1);
      renderizarTablaActual();
      actualizarTotal();
    }
  });
}

La Figura 6 presenta el uso de Swal.fire con un objeto de configuración que define el título, el mensaje, el icono y los botones. La promesa resultante permite ejecutar la eliminación del arreglo con splice y la re-renderización de la tabla sólo si el usuario confirma la acción.

Paso 7: Filtros estilo Excel en el encabezado de la tabla

Se implementó un sistema de filtrado avanzado donde cada columna de la tabla dispone de su propio campo de filtro en una segunda fila del encabezado. Los filtros se aplican en tiempo real usando lógica AND, es decir, sólo se muestran los registros que cumplen simultáneamente con todos los filtros activos.

Figura 7. Filtros inline estilo Excel en el thead de la tabla.

La Figura 7 muestra la segunda fila del encabezado, con fondo gris para diferenciarla de la fila de títulos. Cada filtro opera de manera independiente: el filtro de nombre y semestre responden al evento input, mientras que el filtro de carrera responde al evento change.

La función aplicarFiltros Lee el valor de los tres campos de filtro y aplica la función filter sobre el arreglo global, comparando cada propiedad del objeto con el valor del filtro correspondiente.

function aplicarFiltros() {
  const textoNombre = filtroNombre.value.trim().toLowerCase();
  const textoCarrera = filtroCarrera.value;
  const textoSemestre = filtroSemestre.value.trim().toLowerCase();

  const resultados = estudiantesRegistrados.filter(function (e) {
    const coincideNombre = textoNombre === "" ||
      e.nombre.toLowerCase().includes(textoNombre);
    const coincideCarrera = textoCarrera === "" ||
      e.carrera === textoCarrera;
    const coincideSemestre = textoSemestre === "" ||
      e.semestre.toString().includes(textoSemestre);
    return coincideNombre && coincideCarrera && coincideSemestre;
  });

  renderizarTabla(resultados);
}

El operador lógico && (AND) asegura que un registro sólo se muestre si cumple con todos los filtros que están activos. Si algún filtro está vacío, se evalúa como true para ese criterio, lo que permite la combinación libre de filtros.

Paso 8: Validación de entrada solo con caracteres alfabéticos

Se implementó una validación en tiempo real que impide la entrada de números, símbolos o caracteres especiales en el campo de nombre. Esto se logró mediante una expresión regular que reemplaza los caracteres no válidos cada vez que el usuario presiona una tecla.

Figura 8. Validación de entrada con expresión regular.

const campoNombre = document.getElementById("nombre");

campoNombre.addEventListener("input", function () {
  this.value = this.value.replace(
    /[^a-zA-ZáéíóúÁÉÍÓÚñÑüÜ\s]/g, ""
  );
});

filtroNombre.addEventListener("input", function () {
  this.value = this.value.replace(
    /[^a-zA-ZáéíóúÁÉÍÓÚñÑüÜ\s]/g, ""
  );
  aplicarFiltros();
});

La Figura 8 presenta la expresión regular /[^a-zA-ZáéíóúÁÉÍÓÚñÑüÜ\s]/g, que identifica cualquier carácter que no sea una letra (incluyendo tildes, diéresis y ñ) ni un espacio. El método replace elimina estos caracteres del valor del campo, y el evento input se ejecuta en cada pulsación de tecla, lo que garantiza una validación continua e inmediata.

Toastify se empleó para generar notificaciones temporales en la esquina superior derecha de la pantalla, informando al usuario sobre cada operación exitosa o fallida del sistema.

La Figura 9 muestra la configuración de una notificación Toastify. Las notificaciones de éxito utilizan tonos verdes, las de edición tonos azules, las de eliminación tonos rojos y las de limpieza tonos grises.

Discusión

Los resultados obtenidos en esta práctica corroboran de manera directa los fundamentos teóricos que describen el DOM como una estructura manipulable por programación. Flanagan (2011) plantea que el navegador construye un árbol de objetos a partir del HTML, y yo pude comprobar esta afirmación al observar cómo cada llamada a document.createElement generaba un nodo nuevo que, al insertarse con appendChild, modificaba la apariencia visual de la página sin recargarla. La separación entre la fuente de datos (el arreglo global) y su representación visual (la tabla HTML) resultó ser un patrón fundamental para mantener la integridad de la información y facilitar operaciones como la búsqueda y el filtrado.

La implementación del modo de edición exigió un manejo cuidadoso del estado de la aplicación. En un principio, presentó la dificultad de que al eliminar un estudiante mientras otro se estaba editando, el índice almacenado quedaba apuntando a una posición incorrecta del arreglo. Esta situación se resolvió verificando si el índice de edición coincidía con el índice eliminado, y en caso afirmativo, restableciendo el estado a su valor inicial. Crockford (2008) advierte sobre la complejidad de manejar estados múltiples en aplicaciones JavaScript, y esta experiencia confirmó que un diseño cuidadoso de las variables de estado es esencial para evitar comportamientos inesperados.

La integración de SweetAlert2 y Toastify aportó mejoras sustanciales en la experiencia visual. Las alertas nativas del navegador resultan básicas y poco personalizables, mientras que estas librerías permiten configurar colores, animaciones, iconos y duración de las notificaciones. SweetAlert2 en particular resultó valioso para las confirmaciones críticas como la eliminación de registros, donde un error del usuario podría provocar pérdida de datos. Toastify, por su parte, proporcionó retroalimentación inmediata sobre cada operación, reforzando la percepción de que el sistema responde de manera confiable a las acciones del usuario.

Conclusiones

Desarrollé una aplicación web interactiva para el registro de estudiantes utilizando manipulación del DOM con JavaScript vanilla, construyendo un formulario de entrada, una tabla dinámica, un modal Bootstrap para visualización JSON y un sistema de filtros inline. El trabajo se ejecutó mediante métodos como getElementById, createElement, appendChild y innerHTML, combinados con el manejo de eventos click e input para lograr una interfaz que se actualiza en tiempo real. El propósito de comprender cómo JavaScript interactúa con el DOM se cumplió de manera efectiva, pues logré construir un sistema funcional donde cada acción del usuario provoca una respuesta visual inmediata sin recarga de página. El resultado más significativo fue la comprensión de cómo la separación entre datos y representació visual facilita el desarrollo y el mantenimiento del código.

Implementé un arreglo global de objetos que almacena los registros de estudiantes y permite operaciones completas de CRUD (crear, leer, actualizar, eliminar), validando la entrada del usuario con expresiones regulares y manteniendo la integridad de los datos durante cada operación. La herramienta utilizada fue JavaScript vanilla apoyado en el framework Bootstrap 5.3.3 para el diseño responsivo y las librerías SweetAlert2 y Toastify para las notificaciones visuales. El propósito de determinar la utilidad de los arreglos de objetos como fuente de datos estructurada se alcanzó plenamente, ya que cada funcionalidad del sistema (registro, edición, eliminación, búsqueda y filtrado) opera sobre el mismo arreglo central. Aprendí que la precisión en el manejo de índices y la sincronización entre el estado interno y la vista son aspectos críticos para evitar errores en aplicaciones de este tipo.

Integré librerías externas (SweetAlert2 y Toastify) en el proyecto, evaluando cómo estas herramientas superan las capacidades nativas del navegador para mejorar la experiencia visual del usuario. SweetAlert2 se empleó para las confirmaciones de eliminación y limpieza total, mientras que Toastify generó notificaciones temporales para informar sobre cada operación. El propósito de evaluar la integración de librerías externas y su impacto en la calidad de la interfaz se cumplió al comparar la respuesta visual del sistema antes y después de su implementación. Los resultados muestran una mejora perceptible en la claridad de los mensajes, la estética de las alertas y la rapidez con que el usuario recibe retroalimentación. La lección más valiosa fue comprender que la selección adecuada de herramientas externas puede transformar una aplicación funcional en una aplicación pulida y profesional.

Glosario

Término Definición
DOM (Document Object Model) Representación en forma de árbol del documento HTML que el navegador genera al cargar una página web, donde cada elemento se convierte en un nodo accesible por programación.
Nodo Cada uno de los elementos que componen el árbol del DOM, incluyendo etiquetas HTML, atributos, textos y comentarios.
getElementById Método del objeto document que localiza y retorna un elemento HTML a partir del valor de su atributo id.
createElement Método que genera un nuevo nodo del tipo especificado (div, tr, td, button, etc.) sin insertarlo en el DOM.
appendChild Método que inserta un nodo como hijo al final de un elemento padre, modificando la estructura del árbol DOM.
innerHTML Propiedad que permite leer o escribir el contenido HTML de un elemento. Su uso con cadenas vacías elimina todos los nodos hijos.
textContent Propiedad que permite leer o escribir el contenido textual plano de un elemento, sin interpretar etiquetas HTML.
addEventListener Método que asocia una función (listener) a un evento específico de un elemento, ejecutándola cuando el evento ocurre.
Evento click Señal que se genera cuando el usuario presiona y suelta el botón del ratón sobre un elemento interactivo.
Arreglo (Array) Estructura de datos que almacena una colección ordenada de elementos, accedidos por su posición (índice).
JSON.stringify Método de JavaScript que convierte un objeto o arreglo en una cadena de texto con formato JSON.
Bootstrap Framework CSS de código abierto que proporciona componentes preestilizados para el diseño responsivo de páginas web.
Modal Componente de interfaz que se superpone sobre el contenido principal, requiriendo interacción del usuario para cerrarse.
SweetAlert2 Librería de JavaScript de código abierto que reemplaza las alertas nativas del navegador por ventanas modales estilizadas.
Toastify Librería de JavaScript que genera notificaciones flotantes temporales (toasts) en una esquina de la pantalla.
Expresión regular Patrón de caracteres que define una regla de búsqueda o validación sobre cadenas de texto, utilizado para filtrar o reemplazar caracteres.
CRUD Acrónimo de Create, Read, Update, Delete (Crear, Leer, Actualizar, Eliminar). Representa las operaciones básicas de manipulación de datos.
CDN (Content Delivery Network) Red de distribución de contenido que aloja bibliotecas y frameworks en servidores globales, permitiendo su carga rápida desde el navegador.

Resumen Ejecutivo

La práctica de laboratorio abordó el desarrollo de una aplicación web para el registro dinámico de estudiantes, empleando manipulación del DOM mediante JavaScript vanilla. El sistema implementó un formulario de entrada con validación alfabética, una tabla dinámica con acciones por fila (ver JSON, editar, eliminar), un modal Bootstrap para la visualización de datos serializados en formato JSON, filtros inline estilo Excel por columna y la integración de SweetAlert2 y Toastify para mejorar la retroalimentación visual. El trabajo permitió consolidar conocimientos sobre acceso a elementos HTML, creación y modificación de nodos, manejo de eventos, uso de arreglos de objetos como fuente de datos y presentación de informacióon estructurada. Los resultados demuestran que la combinación de fundamentos teóricos del DOM con librerías externas produce interfaces funcionales, pulidas y con una experiencia de usuario significativamente superior a las soluciones nativas del navegador.
